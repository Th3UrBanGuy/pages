The "Async Polling" Architecture

Client
Hono API
Redis Queue
Worker
MinIO S3

To bypass the 504 Gateway Timeout imposed by proxies on long requests (10s-120s), we implemented a BullMQ job queue. The API immediately returns a 202 Accepted with a Job ID, allowing the client to poll for status while the heavy lifting happens in the background.

Challenges Conquered

Self-Hosted S3

Integrated MinIO within the Docker ecosystem to simulate cloud storage locally. Configured signed URLs for secure file access.

CI/CD Pipeline

Automated workflow that runs ESLint, Prettier, and E2E Tests on every push. Blocks deployment if tests fail.

Bonus: Observability

Full implementation of OpenTelemetry with Jaeger for distributed tracing and Sentry for error tracking.

Bonus: React Dashboard

Built a Next.js frontend to visualize the background job status, download progress, and real-time system health.
